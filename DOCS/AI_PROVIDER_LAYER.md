# AI Provider Layer — сменяемые ИИ-провайдеры (PR-8)

Независимый слой поставщиков ИИ: бизнес-логика не привязана к одному SDK.
Администратор выбирает провайдера и модель для каждого агента, с резервным
провайдером на случай недоступности основного.

## Модель данных (миграция 0045)

- `ai_providers` — подключённые поставщики (`openai`/`anthropic`/`gemini`/`local`):
  название, включён, base_url, модель по умолчанию, признак настроенного ключа
  (`credentials_configured_externally`).
- `ai_models` — модели провайдера (поддержка изображений/инструментов, лимит токенов).
- `agent_ai_assignments` — назначение агенту: основной/резервный провайдер и модель,
  температура, уровень рассуждения, лимит токенов, месячный бюджет, тайм-аут,
  разрешения на изображения/документы/инструменты.
- `ai_usage_records` — метаданные расхода (токены, стоимость, request_id). **Без
  промптов и скрытых рассуждений** (CLAUDE.md §13, §15).
- `ai_provider_health` — состояние подключения (`unknown`/`ok`/`down`).

## Слой адаптеров (`app/services/ai_provider.py`)

Единый контракт `AIProviderAdapter.generate(prompt, model, params) -> AIResult`
(с `usage`). Адаптеры: `OpenAIAdapter`, `AnthropicAdapter`, `GeminiAdapter`,
`LocalOpenAICompatibleAdapter`. HTTP-транспорт внедряется (тесты без сети).

- **Реальные вызовы по умолчанию ВЫКЛЮЧЕНЫ** (`AI_REAL_CALLS=false`) — работает
  безопасный **эхо/заглушка**-режим для разработки.
- `run_for_agent(...)` — основной провайдер, при недоступности → резервный
  (fallback); пишет расход без промптов.
- `check_health(...)` — проверка подключения без раскрытия ключа.

## Безопасность ключей (CLAUDE.md §13, §19)

- Ключи доступа читаются **только из окружения/secret manager**
  (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GEMINI_API_KEY`, `LOCAL_AI_*`).
- **Никогда не хранятся в БД** в открытом виде и не в Git.
- В интерфейсе и API — только маскированный индикатор (`mask_secret`), сам ключ
  не возвращается.

## API `/ai-providers`

Список/создание провайдеров, включение/отключение, проверка подключения
(`/health`), модели провайдера, назначение модели агенту
(`PUT /agents/{id}/assignment`), расход (`/usage`). RBAC: `ai.provider.manage`
(администрирование), `ai.provider.view` (просмотр).

## Экран

«Настройки → ИИ-провайдеры»: подключение провайдера, модель по умолчанию,
включение/отключение, проверка подключения, назначение модели агентам, расход;
ключи маскируются.

## Требует решения специалистов ООО «Экстра-Элит»

- Реальные API-ключи провайдеров (задаёт заказчик через окружение) и включение
  `AI_REAL_CALLS=true`.
- Актуальные тарифы для точного расчёта стоимости (сейчас — грубая демо-оценка).
- ИИ формирует только предложения и черновики; окончательные юридические,
  финансовые, кадровые, нормативные и внешние действия — за уполномоченным
  человеком.
