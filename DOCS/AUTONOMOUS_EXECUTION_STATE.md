# AUTONOMOUS_EXECUTION_STATE.md — состояние автономного выполнения

**Назначение:** отслеживать прогресс автономной реализации дорожной карты
Badrudin AI OS. Если сессия прервана, следующая сессия читает этот файл и
продолжает с первого незавершённого этапа.

**Режим:** автономный (слияние PR при зелёном CI без запроса подтверждения).
Условия обязательной остановки и запрета слияния — см. требования пользователя
(красный/выполняющийся CI, конфликт миграций, потеря данных, секрет, нарушение
tenant isolation, ошибки RBAC/ABAC, доступ к чужим файлам, самоутверждение
критического результата автором).

**Не выполнять:** реальные внешние отправки; секреты в репозитории; окончательное
утверждение решением ИИ; ослабление проверок безопасности.

## Статус: демо-дорожная карта выполнена

Все запланированные этапы (#39–#53) слиты в `main`. Автономный режим завершён.
Сквозной демонстрационный сценарий процессного ядра проверен на чистой БД
(`backend/scripts/demo_scenario.py`): директор ставит процесс → согласование R3
независимым руководителем → РП назначает инженера → инженер принимает и стартует →
прикладывает фото и акт (Evidence Gate) → ИИ-черновик отчёта (предложение, не
утверждение) → подтверждение человеком → отправка на проверку → возврат на
доработку → повторная отправка → закрытие независимым проверяющим. Все инварианты
(SoD, Evidence Gate, честный ИИ-контур) соблюдены.

## Известные ограничения / требует решения специалистов ООО «Экстра-Элит»

- Юридическая применимость и актуальная редакция нормативных документов —
  подтверждает уполномоченный специалист (главный инженер/ПТО/юрист); система по
  умолчанию ставит статус `needs_review`.
- Оператор квалифицированной электронной подписи — не выбран (не входит в контур).
- Реальные ключи внешних сервисов (email/Telegram/WhatsApp/СМС) — отсутствуют;
  внешняя отправка не выполняется (только внутренний контур `in_app`).
- Реальный анализ фото/видео моделью ИИ — вне backend-контура; ИИ формирует
  черновики из фактических данных и не выносит окончательных решений.
